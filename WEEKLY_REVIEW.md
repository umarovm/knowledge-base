# WEEKLY_REVIEW — еженедельный разбор

_Триггер: пользователь просит «недельный разбор», «разобрать проекты» и т.п.
Claude читает этот файл и ведёт сессию по процедуре ниже._

## Состояние

- Последний разбор: — (ещё не проводился)
- Порядок проектов: job-search → work → finance → projects → learning → ideas → health

## Процедура

**Формат: интерактивно.** Один проект за раз; после дайджеста — дождаться комментариев
пользователя, вместе сформулировать задачи, и только потом переходить к следующему проекту.
Не разбирать всё молча и не приносить готовый отчёт.

### Шаг 0. Подготовка

1. Прочитай запись последней сессии в логе ниже (контекст: что решено, что отложено).
2. `inbox/` не пуст → сначала разбери его по процедуре из корневого CLAUDE.md.
3. Запусти дайджест: `python3 scripts/review_digest.py`
   Он покажет по каждому проекту: новые заметки с последнего разбора (по `date:`),
   свежие строки логов living-файлов, открытые/просроченные задачи, `related:`-связи.
   Тела заметок читай только когда summary из дайджеста недостаточно для обсуждения.

### Шаг 1. По каждому проекту (в порядке из шапки)

1. Покажи пользователю дайджест проекта: что нового, что зависло (просрочка,
   задачи без движения), какие мосты `related:` ведут в другие папки.
2. По мостам подтяни 1–2 связанных файла из других папок, если они нужны для сути —
   не сканируй чужие папки целиком.
3. Обсуди с пользователем: его выводы, прогресс, новые мысли.
4. Итоги сразу зафиксируй:
   - новые задачи → SMART-формулировка → в файл-хозяин (living-файл или PLAN.md);
   - просроченные задачи → спроси «закрыть / перенести / переформулировать?» —
     не закрывай сам;
   - новое решение/мысль → заметка по обычным правилам; событие → лог living-файла.
5. Переходи к следующему проекту только после подтверждения пользователя.

### Шаг 2. Обзор и завершение

1. Пройди секцию «Заметки для еженедельного обзора» корневого index.md:
   по каждой — решение (действие / в проект / оставить / архив). Решённые убери из секции.
2. Секция дайджеста «Идеи без движения 30+ дней»: по каждой — SMART-действие,
   перенос в проект или `status: archived`. «Оставить полежать» — не вариант.
3. Сверка с PLAN.md: вехи и недельный ритм — что закрыто за неделю, что смещается.
   Изменения вех — в PLAN.md.
4. Запусти `python3 scripts/check.py`; найденное исправь (спорное — спроси пользователя).
5. Запусти `python3 scripts/rebuild_indexes.py`.
6. Обнови шапку этого файла (дата разбора) и добавь запись в лог: 2–3 строки —
   главные решения и что отложено. Без пересказа заметок (владение фактами).
