## Процедура: глубокий разбор одного проекта

0. `inbox/` не пуст → сначала разбери его по `inbox/CLAUDE.md`
   (иначе картина проекта неполная).
1. Контекст: записи лога ниже по этому проекту с прошлых сессий.
2. Прочитай проект **целиком**: index.md, затем тела всех заметок и living-файлов,
   включая подпапки (`_ARCHIVE` не читать). Исключение — файлы-сырьё (транскрипты
   интервью, экспорты, всё заметно крупнее обычной заметки): целиком не читать,
   в карту включить по названию и summary, открывать выборочно и только если
   обсуждение этого требует. По секции «Связи» index.md подтяни 1–2 файла
   из других папок, если они нужны для сути.
3. Выдай один отчёт:
   - **Карта**: каждый файл одной строкой — что говорит, дата, статус.
   - **Зависшее**: просроченные задачи, задачи и заметки без движения,
     протухшие шапки living-файлов.
   - **Предложения** одной таблицей: файл → действие (SMART-задача / merge /
     архив / удалить / обновить шапку / вынести в подпапку / добавить `related:`)
     + обоснование одной фразой. Спорное помечай «?».
4. Обсуждение по пунктам. Исполняй только явно подтверждённое («всё ок» = вся
   таблица); удаление файла — только при явном подтверждении этой строки.
5. Исполни разом: задачи — SMART-формулировкой в файлы-хозяева, merge/архив/шапки.
   Затем `python3 scripts/rebuild_indexes.py` и `python3 scripts/check.py`.
6. Обнови дату проекта в таблице выше; строка в лог — 2–3 итога сессии,
   без пересказа заметок (владение фактами).

  решённые пункты убраны.

  (быстрый срез нового по всем папкам, без чтения тел).
