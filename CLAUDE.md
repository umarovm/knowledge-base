# knowledge-base

Личная база знаний. Заметки приходят голосом через Telegram-бот в `inbox/`,
затем сортируются по проектным папкам.

## Правила для агента

- Не выдумывай факты о пользователе. Источник истины — сами заметки.
- Не читай заметки «вслепую»: сначала `index.md` нужного проекта, тела файлов — только при необходимости.
- Любое добавление/перемещение/переименование заметки = обновление `index.md` этого проекта.
- Имена файлов: `YYYY-MM-DD-slug.md`. Каждая заметка начинается с YAML frontmatter
  (`date, source, project, tags, status, summary`).
- `status`: raw → processed → archived.
- `inbox/` — append-only для бота; сортировка выполняется здесь, в Cowork.

## Проекты

- `work/` — текущая работа (Revolut): проекты, promotion, рабочие заметки
- `job-search/` — поиск работы: CV, интервью, drills
- `projects/` — личные проекты (подпапка на проект)
- `finance/` — личные финансы, бюджет, инвестиции
- `health/` — здоровье, спорт, сон
- `learning/` — обучение, курсы, книги
- `ideas/` — идеи, которые пока не привязаны к проекту
- `captures/` — сырые транскрипции бота (не трогать)

В корне: `ABOUT_ME.md` (контекст о пользователе) и `PLAN.md` (стратегический план) —
читать, когда нужен личный контекст для советов.

Детали и состояние проектов — в `index.md`.
