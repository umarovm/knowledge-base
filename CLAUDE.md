## Базовые правила (действуют во всех папках)

- Не выдумывай факты о пользователе. Источник истины — сами заметки.
- Не читай заметки «вслепую»: сначала `index.md` нужной папки, тела файлов — только при необходимости.
- Любое добавление/перемещение/переименование заметки = обновление `index.md` этой папки
  и, если менялся состав папки, корневого `index.md`.
- Имена файлов: `YYYY-MM-DD-slug.md`. Слаг — по-русски, коротко, без времени и tg-суффиксов.
- `inbox/` — append-only для бота. `captures/` — сырые транскрипции, не трогать.

## Frontmatter (канонический формат)

```yaml
date: YYYY-MM-DD           # дата заметки
source: "[[captures/...]]" # ссылка на исходный capture (от бота)
capture_id: "tg-..."       # сохранять при разборе
project: finance           # папка назначения
tags: [investing, crypto]  # из словаря тегов ниже
status: processed          # raw | processed | archived
summary: "1–2 предложения"
related:                   # опционально: мосты к заметкам других папок
  - "[[work/2026-07-15-реквизишн-тимлид-и-ночные-смены]]"
```

Бот пишет свой формат (`created, capture_id, source, destination`) — при разборе
конвертировать в канонический.

Статусы: `raw` — как пришло от бота; `processed` — отсортировано, канонический
frontmatter; `archived` — неактуально, хранится для истории.

## Мосты между проектами

Темы пересекаются (зарплата в work влияет на депозитный план в finance и т.п.).
Чтобы получать контекст без чтения чужих папок целиком:

- Если заметка затрагивает другую папку — добавь `related:` в её frontmatter
  и по строке в секцию «Связи» в `index.md` ОБЕИХ папок: ссылка + 1 строка, почему связано.
- Работая над вопросом, сначала проверь секцию «Связи» в index.md текущей папки —
  она укажет, какие 1–2 файла из других папок стоит прочитать. Не сканируй чужие папки.

## Маршрутизация inbox

| Тема | Папка |
|---|---|
| Revolut: проекты, promotion, смены, зарплата | `work/` |
| CV, интервью, отклики, drills | `job-search/` |
| Инвестиции, крипта, депозиты, бюджет, налоги | `finance/` |
| Спорт, сон, питание, врачи | `health/` |
| Курсы, книги, конспекты | `learning/` |
| Активный личный проект | `projects/<имя>/` |
| Идея без проекта и без действий по ней | `ideas/` |

Спорные случаи:
- Инвестиционная стратегия → `finance/`, даже если упоминается работа или переезд.
- Идея нового проекта → `ideas/`, пока по ней нет реальных действий; начал делать → `projects/<имя>/`.
- Зарплата/смены на текущей работе → `work/`, финансовый расчёт из той же заметки → `finance/`
  (разбить на две + связать через `related:`).
- Не уверен → `ideas/` + пометка в «Заметки для еженедельного обзора» в корневом `index.md`.

## Процедура разбора inbox

1. Прочитай все заметки inbox (кроме CLAUDE.md).
2. Заметки одного `capture_id` с пересекающимся содержанием — объединяй в одну
   (бот дробит одну голосовую на куски).
3. Определи папку по таблице маршрутизации.
4. Переименуй по конвенции, перепиши frontmatter в канонический формат.
5. Затрагивает другие папки → проставь мосты (`related:` + «Связи» в index.md обеих папок).
6. Перемести, обнови `index.md` папки и корневой `index.md`.
7. Inbox после разбора пуст (кроме CLAUDE.md).

## Структура и тематические подпапки

Папки и их состояние — в корневом `index.md`. Каждая папка: свой `CLAUDE.md`
(границы) + `index.md` (каталог + связи). У проектов в `projects/` — подпапка
на проект со своим `index.md`.

Чтобы заметки не копились плоской кучей:

- Набралось 3+ заметок на одну тему внутри папки → создай тематическую подпапку
  (например `finance/инвестиции/`) со своим `index.md` и перенеси их туда.
- Родительский `index.md` перечисляет подпапки в секции «Подпапки» одной строкой
  (тема, что внутри, число заметок) — без дублирования содержимого их индексов.
- Пока заметок по теме меньше 3 — они лежат плоско в корне папки.
- При переносе в подпапку обнови `project:` во frontmatter и все wiki-ссылки на заметку.
- Новая заметка при разборе inbox кладётся сразу в подходящую подпапку, если она есть.
