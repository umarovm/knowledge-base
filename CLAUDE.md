## Владение фактами (анти-дубликация)

У каждого факта один хозяин, остальное — производное. Никто не пересказывает чужое,
только ссылается:

- **Заметка** владеет решением/событием и его обоснованием.
- **Шапка living-файла** владеет текущим состоянием (числа, статусы).
- **Frontmatter** владеет метаданными и связями (`related:`).
- **Все index.md, TODO.md и themes/ — генерируются** скриптом `scripts/rebuild_indexes.py` из frontmatter.
  `themes/<тег>.md` — сквозные темы (список тегов — `THEME_TAGS` в скрипте): вся картина
  по теме через все папки одним файлом.
  Руками их не редактировать (исключение: секция «Заметки для еженедельного обзора»
  в корневом index.md). После любого изменения заметок запускай:
  `python3 scripts/rebuild_indexes.py`.
- Дайджест «Обо мне» ниже синхронизируется только при изменении `ABOUT_ME.md`.

## Еженедельный разбор

Пользователь просит «недельный разбор», «разобрать проекты», «weekly review» →
прочитай `WEEKLY_REVIEW.md` и веди сессию строго по процедуре оттуда
(старт — `python3 scripts/review_digest.py`, папки заранее не сканировать).

## Базовые правила (действуют во всех папках)

- Не выдумывай факты о пользователе. Источник истины — сами заметки.
- Не читай заметки «вслепую»: сначала `index.md` нужной папки, тела файлов — только при необходимости.
- Любое добавление/перемещение/переименование заметки = запуск `scripts/rebuild_indexes.py`.
- `scripts/check.py` — валидатор (ссылки, frontmatter, теги, задачи в таблицах).
  Запускать после больших правок; находки исправлять, спорные — спрашивать.
- `templates/` — шаблоны жанров (конспект, решение, идея). Новую заметку узнаваемого
  жанра — писать по шаблону; при разборе inbox приводить тело к шаблону, если жанр очевиден.
- `summary` во frontmatter — самодостаточный: 1–2 предложения с конкретикой (числа,
  решения), чтобы принимать решение о заметке, не открывая её. Не «мысли про X».
- `inbox/` — append-only для бота; разбор — по `inbox/CLAUDE.md`. `captures/` — сырые
  транскрипции, не трогать.

## Мосты между проектами

Темы пересекаются (зарплата в work влияет на депозитный план в finance и т.п.).
Чтобы получать контекст без чтения чужих папок целиком:

- Если заметка затрагивает другую папку — добавь `related:` в её frontmatter.
  Это единственное место, где связь пишется руками: секции «Связи» в index.md
  обеих папок генерирует скрипт.
- Работая над вопросом, сначала проверь секцию «Связи» в index.md текущей папки —
  она укажет, какие 1–2 файла из других папок стоит прочитать. Не сканируй чужие папки.

## Living-файлы (мейнтейнит Claude)

Помимо заметок-событий в папках живут файлы состояния (`type: living` во frontmatter,
имя CAPS: `PORTFOLIO.md`, `PROMOTION.md`, …). Устройство: **шапка** = текущее состояние
(перезаписывается), **лог** = append-only строки событий. Читай шапку, не перечитывай лог.

- Заметка из inbox описывает повторяющееся событие для существующего living-файла
  (дрилл, отклик, сессия практики, изменение портфеля, статус реквизишна) →
  строка в его лог + обновление шапки, отдельная заметка НЕ создаётся
  (сырая транскрипция и так остаётся в captures/).
- Заметка содержит и событие, и новое решение/мысль → лог обновить И создать заметку.
- Набирается новый устойчивый поток однотипных событий без своего living-файла →
  создай его по этому же шаблону (шапка + лог), зарегистрируй в index.md папки.
- Living-файлы перечислены в index.md своей папки со статусом `living`.
- Лог длиннее ~30 строк → старейшие строки перенести в `<ИМЯ>_ARCHIVE.md` того же
  формата со `status: archived` (скрипты и Claude его не читают). `check.py` напомнит.

Текущие: finance/PORTFOLIO.md · work/PROMOTION.md · job-search/APPLICATIONS.md ·
job-search/STORY_BANK.md · job-search/INTERVIEW_DRILLS.md · learning/SKILLS_LOG.md ·
projects/wikibot/BACKLOG.md · projects/трейдинг-боты/BACKLOG.md.

## Задачи (to-do)

- Открытая задача = строка вида `- [ ] текст 📅 YYYY-MM-DD` (формат плагина Obsidian Tasks).
  Дата опциональна; повторяющиеся задачи — `🔁 every week` вместо даты. В таблицах задачи
  не писать — только списочными строками, иначе скрипт и плагин их не увидят.
- Хозяева задач: living-файлы (обычно секция «Задачи» или «Открыто») и `PLAN.md`
  (вехи, недельный ритм). Одна задача — один хозяин, не дублировать.
- Корневой `TODO.md` — **генерируется** `scripts/rebuild_indexes.py` из этих строк
  (просрочено / по дедлайнам / еженедельный ритм / без даты). Руками не редактировать.
- Закрытие задачи: `[x]` в файле-хозяине (при необходимости — строка в лог) → запустить скрипт.
- Living-файлы со `status: archived` скриптом не сканируются; их чекбоксы писать как `- ☐`.
- Задача-мысль по идее без действий → не задача, а пометка в «Заметки для еженедельного
  обзора» корневого index.md.
- **Формулировка — SMART.** Любая новая задача или веха (из обсуждения, разбора inbox,
  правки PLAN) записывается так: конкретный результат («что будет готово», а не
  «поработать над»), измеримый критерий (число / артефакт / kill-критерий), реалистичный
  объём, привязка к цели файла-хозяина, срок `📅` — если он есть по смыслу.
  Плохо: «- [ ] подтянуть SQL». Хорошо: «- [ ] 10 задач на window functions с устным
  объяснением 📅 2026-08-01». Расплывчатую формулировку при разборе переписывать в SMART;
  если измеримого действия нет — это не задача, а идея/пометка в обзор.

## Фиксация решений из сессий

Содержательная сессия с Claude закончилась решениями, новыми задачами или фактами →
перед завершением Claude сам предлагает план фиксации одной таблицей (как при разборе
inbox): что → новая заметка / строка в лог living-файла / SMART-задача в файл-хозяин.
После подтверждения — исполнить и запустить `scripts/rebuild_indexes.py`.
Принцип: не зафиксировано в базе — значит, не существует.

## Структура и тематические подпапки

Папки и их состояние — в корневом `index.md`. Каждая папка: свой `CLAUDE.md`
(границы) + `index.md` (каталог + связи). У проектов в `projects/` — подпапка
на проект со своим `index.md`.

Чтобы заметки не копились плоской кучей:

- Набралось 3+ заметок на одну тему внутри папки → создай тематическую подпапку
  (например `finance/инвестиции/`) со своим `index.md` и перенеси их туда.
- Родительский `index.md` перечисляет подпапки в секции «Подпапки» одной строкой
  (тема, что внутри, число заметок) — без дублирования содержимого их индексов.
- Пока заметок по теме меньше 3 — они лежат плоско в корне папки.
- При переносе в подпапку обнови `project:` во frontmatter и все wiki-ссылки на заметку.
- Новая заметка при разборе inbox кладётся сразу в подходящую подпапку, если она есть.
